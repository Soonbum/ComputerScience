# 컴퓨터 과학

* 본 글은 컴퓨터 과학(Computer Science)을 가장 기본적인 개념부터 이해할 수 있도록 정리한 글입니다. (저자: 정순범)

## 컴퓨터 과학 개론

### 컴퓨터의 역사

* 컴퓨터의 시초는 주판과 같이 계산 보조 도구로 시작해서 기계식 계산기, 그리고 현재 널리 사용하는 전자식 계산기까지 나아왔습니다.
  - 이제는 컴퓨터가 본래 목적인 계산을 위한 도구로서만이 아니라 인류가 요구하는 다양한 분야의 업무를 보조해주는 범용적인 도구로 사용되고 있습니다.
  - 어떻게 보면 자동 계산기에 불과한 이 물건(?)이 어떤 원리로 사람들의 많은 일들을 도와주는 유익한 도구가 될 수 있었을까요?
  - 컴퓨터의 역사와 발전 방향을 보면 컴퓨터가 마법 상자가 아니라는 것을 알 수 있게 되고 컴퓨터에 대한 폭넓은 이해를 얻게 될 것입니다.

 * 컴퓨터는 인류가 자동화에 집착하면서 발전했다고 봐도 됩니다.
   - 주판
   - 기계식 계산기: 암산하지 않고도 돌리면 자동으로 계산해준다.
   - 천공카드와 직조기: 천공카드 패턴에 맞게 복잡한 직조 패턴이 결정된다.
   - 차분기관/해석기관: 더욱 복잡한 기계식 계산기
   - 전자 계산기 이론에서 제시하는 최초의 프로그램 내장 방식 컴퓨터: 콜로서스, 에니악 등의 등장
   - 기계식 계전기, 진공관, 그리고 트랜지스터의 등장: 소자 발견으로 인해 컴퓨터의 소형화가 빠른 속도로 이루어진다.

* 전자식 컴퓨터의 등장은 소자의 발견으로 인해 가능했습니다.
  - 계전기, 진공관, 트랜지스터 모두 작동 원리는 동일합니다. (시간이 갈수록 고장은 덜 나고 크기는 작아지고 전력 소모는 작아지면서 집적도가 고도로 높아짐)
  - 이것들은 전기를 흐르게 하거나 막히게 하는 스위치라고 할 수 있습니다. 다만 일반 스위치와 다른 점이 있다면 이것을 손으로 켜고 끄는 게 아니라 다른 전기선을 이용해 켜고 끄는 역할을 합니다.
  - 전기 스위치로서 역할을 하는 원리에 대해서는 복잡하기 때문에 생략합니다. (계전기, 진공관, 트랜지스터의 작동 원리를 검색해 보세요)
  - 초창기 반도체 소자(CPU 및 메모리 등)는 수많은 트랜지스터 및 도선을 직접 wiring 했으나 집적도가 고도로 높아지면서 에칭(Etching, 식각) 공정으로 사진 찍듯이 패턴을 새기는 식으로 제작하고 있습니다.
  - 또한 도선으로 직접 연결하는 방식에서 PCB 기판 등을 통해 패키징하여 더욱 집적도를 높이는 식으로 발전해 왔습니다.
  - 현재는 빛의 파장 크기만큼이나 도선이 가늘어지고 양자 터널링에 의해 누설 전류도 증가하고 있어서 집적도를 높이는 것의 한계에 이른 상황입니다.

### 컴퓨터 작동 원리

* 이런 전기 스위치를 가지고 무엇을 할 수 있을까요?
  - 2가지 상태를 표현하는 수를 2진수라고 합니다. (0부터 9까지는 10진수)
  - 2진수를 사용하는 이유는 노이즈에 강하고 오류가 적고 교정하기도 쉽기 때문입니다.
  - 독일 수학자 조지 부울이라는 사람이 제안한 논리 연산을 기반으로 한 수학이 있습니다.
  - 논리 연산의 가장 기초는 AND, OR, NOT이 있는데 이것은 전기 스위치로 다 구현할 수 있습니다. 그리고 논리 연산을 조합하면 모든 연산이 가능해집니다.
    * AND 게이트: 2개의 전기 스위치를 직렬로 연결하고 둘 다 ON 상태로 하면 전기가 흐릅니다.
    * OR 게이트: 2개의 전기 스위치를 병렬로 연결하고 하나만 ON 상태로 하면 전기가 흐릅니다.
    * NOT 게이트: 1개의 전기 스위치의 입력부를 도선 중간에 달고 출력부를 접지(GND)에 연결하면 ON 상태가 되면 전기가 끊어지고, OFF 상태가 되면 전기가 흐릅니다.
  - 이제 가장 기본적인 3가지 게이트를 이용해서 새로운 논리 게이트를 만들 수 있습니다.
    * XOR 게이트: 병렬로 연결된 AND + NOT 게이트와 OR 게이트의 출력이 AND 게이트의 입력으로 들어가는 구조 (한 쪽만 ON 상태일 때 전기가 흐름)
    * 반가산기: XOR 게이트와 AND 게이트가 병렬로 연결되어 있으며 1자리수 덧셈을 할 수 있습니다.
    * 반가산기 2개와 OR 게이트를 연쇄적으로 연결하면 전가산기가 되며 이것이 여러 개가 있으면 n자리수 덧셈을 할 수 있습니다. (예를 들어, 8자리수 연산을 할 수 있으면 8비트 연산기)
    * 덧셈만 가능한 트랜지스터로 사칙연산을 어떻게 구현하나요? 뺄셈은 2의 보수 연산을 하면 됩니다. 곱셉은 덧셈의 반복, 나눗셈은 뺄셈의 반복입니다.
    * 이런 식으로 뺄셈을 위한 감산기, 그 외에도 곱셈, 나눗셈, 기타 여러 가지 연산(실수 연산, 병렬 연산)을 위한 것들이 계속 추가되면 범용 계산기가 됩니다.
  - 현재 주로 사용하는 64비트 컴퓨터란 연산 단위 및 레지스터 크기가 64비트이며 접근할 수 있는 메모리 주소가 64비트임을 의미합니다. 또한 CPU와 메모리 간의 데이터 전송 통로인 데이터 버스 역시 64비트 처리가 가능합니다.

* 연산만 하고 그것을 기억해야만 비로소 온전한 계산기의 역할을 할 수 있는데 그때 필요한 것이 메모리입니다.
  - AND, OR, NOT 게이트를 잘 조합하면 래치라고 부르는 1비트 메모리를 만들 수 있습니다.
  - 래치 여러 개를 합치면 CPU 안에서 계산 결과를 저장하는 레지스터가 됩니다.
  - CPU의 계산 결과를 직접 담는 1차 메모리를 레지스터라고 합니다.
  - 그리고 보조기억장치로부터 읽어온 정보를 담는 DRAM으로 된 주기억장치를 통칭 "메모리"라고 합니다.
  - 레지스터와 메모리 사이에 자주 읽어올 만한 부분을 예측해서 미리 가져와 저장하는 SRAM으로 된 메모리를 "캐시 메모리"라고 합니다.
  - 메모리 회로에는 Read Enable, Write Enable, Input, Output 와이어가 존재합니다. (읽고 싶을 때 읽고, 쓰고 싶을 때 쓰고 값을 설정하거나 읽기 위해 필요함)
  - 멀티플렉서는 Address 와이어가 있어서 2차원으로 배치된 메모리 배열에서 원하는 위치의 메모리 값을 읽거나 쓸 수 있게 해줍니다. (캐시메모리, 메인메모리에 적용됨)

* 메모리의 차이
  - 레지스터/SRAM은 트랜지스터로 구현된 래치를 통해 데이터를 저장하는 반면, DRAM은 캐퍼시터를 이용해 데이터를 저장함
  - 레지스터/SRAM은 집적도가 낮지만 전기가 유지되는 동안 데이터가 유지되고 매우 빠르지만 비싼 메모리
  - DRAM은 집적도가 높고 저렴하지만 데이터를 읽을 때마다, 혹은 주기적으로 캐퍼시터를 충전하는 Refresh를 수행해야 하며 속도가 상대적으로 많이 느림
  - 현대 컴퓨터는 레지스터, 1/2/3차 캐시, DRAM 메인 메모리 순서대로 계층을 두어 가성비를 갖추고 있음

* CPU 안에는 연산을 하는 장치와 여러 개의 레지스터들로 구성되어 있습니다.
  - 연산 장치를 ALU(Arithmetic & Logic Unit)이라고 부릅니다.
    * 사칙연산과 같은 산술 연산과 true, false 결과를 도출하는 논리 연산을 수행함
    * ALU는 연산자(Operation Code)와 피연산자 1개 혹은 2개를 입력 받고 결과를 내보내며 경우에 따라 플래그 레지스터의 값을 설정함
    * ALU의 동작은 순수하게 전기적 스위치에 의해 이루어지며 연산자(Opcode) 또한 사전에 정의된 스위치 패턴에 불과함
  - 연산 동작의 순서 (이것은 기본 개념이며 [어셈블리어](https://github.com/Soonbum/AssemblyLanguage)를 통해 CPU에게 직접 명령을 내릴 수 있음, 흔히 폰 노이만 구조라고 불림)
    * LOAD: 메모리로부터 값을 가져오는 동작 (연산자/피연산자 같이 들어 있음)
    * DECODE: 메모리에서 가져온 값을 해독함
    * EXECUTE: 해독한 대로 연산을 수행함 (레지스터에 결과 값을 저장하고 플래그를 설정함)
    * STORE: 메모리에 값을 저장하는 동작
  - 레지스터의 종류
    * 명령어 레지스터: 수행해야 하는 명령어(연산자)가 들어 있음 (이 값이 연산 장치에 있는 게이트들의 입력 값으로 들어가서 연산을 수행함)
    * 명령어 주소 레지스터: 명령어가 담긴 메모리 주소가 들어 있음 (프로그램 카운터라고도 함)
    * 플래그 레지스터: 현재 CPU의 상태를 저장함
    * 범용 레지스터: 연산하기 위한 값(피연산자)을 저장함
  - 그리고 CPU와 메모리 간의 모든 동작은 "석영"이라는 광물로 만들어진 클럭 생성기를 통해 매우 빠른 속도로 이루어집니다.
    * 클럭이 높아질수록 연산 속도는 빨라집니다. (오버클러킹)
    * 그러나 열이 많이 발생하고 전기도 많이 소모됩니다.
    * 클럭을 낮추면 연산 속도는 느려지지만 전기 소모량과 발열량이 낮아짐
  - 클럭을 높이는 것에도 한계가 있어서 이제는 파이프라이닝, 슈퍼스칼라 같은 기법을 통해서도 속도를 높입니다.
    * 파이프라이닝: 명령어 수행이 다른 결과 값에 의존하지 않을 경우 비순차적으로 명령어를 실행하여 CPU 유휴시간을 줄여서 처리속도를 증가시킴
    * 슈퍼스칼라: 명령어를 처리하는 라인을 늘려서 명령어를 병렬처리 (이 역시 명령어 의존성을 피해야 함)
    * CPU에서 자체적으로 명령어들을 비순차적 실행하는 것도 한계가 있기 때문에 현대에는 컴파일러에 의한 최적화를 통해 처리 속도를 높이고 있음

* 프로그램 언어의 발전
  - 기계어는 0과 1로만 되어 있으며 초창기 컴퓨터는 천공카드, 테이프를 통해 종이에 뚫려 있는 패턴으로 0과 1을 구분하고 그것에 따라 전기 스위치를 조작하여 연산하게 되어 있었습니다.
  - 어셈블리어는 기계어를 좀 더 쉽게 사용할 수 있게 창안된 언어이며 기계어와 1:1 매칭이 됩니다. (어셈블러는 어셈블리어를 기계어로 번역해줌)
  - C 언어와 같은 고급언어들은 컴파일러를 통해 기계어로 번역됩니다.
  - 최초의 컴파일러는 단순한 어셈블러에 불과했고 지원되는 문법도 매우 제한적이었지만, 컴파일러로 컴파일러 코드를 컴파일하는 식(Bootstrapping)으로 발전시켜 왔습니다.
  - 어셈블리어는 기본적으로 산술/논리/비트 연산, 분기 처리, 스택 관리가 가능합니다.
  - 구조체, 클래스, 자료구조, 알고리즘과 같은 개념들은 코드를 통해 구현할 수 있으며 고급 언어들은 간단한 문법만으로 해당 기계어를 생성하도록 컴파일러가 변환해줍니다.

* 컴퓨터로 단 하나의 프로그램, 단 하나의 작업만 하지는 않습니다.
  - 여러 프로그램을 작동시키고, 여러 작업을 하려면 프로그램을 쉽게 실행해주고 스케줄링을 관리해주는 소프트웨어가 필요한데 그것을 운영체제(Operating System)이라고 부릅니다.
  - 운영체제가 존재하기 전에 컴퓨터는 그저 거대한 스위칭 머신에 불과하고 1개의 프로그램만 한 번씩 실행할 수 있었습니다.
  - 운영체제가 있음으로 인해 컴퓨터는 스스로 작동할 수 있게 되었고 사용자가 요청할 때마다 프로그램을 알아서 작동시켜 줄 수 있게 되었습니다. (멀티태스킹)
  - 현대 운영체제는 여러 프로세스가 끊임없이(Seamless) 실행되도록 문맥 교환(Context Switching)을 수행합니다. 또한 악의적인 프로그램으로부터 최소한의 보호 장치가 있습니다. (메모리 경계 보호, CPU 특권 명령 제한, 잘못된 명령에 대한 예외 발생 등)
  - 문맥 교환은 내부적으로 CPU가 기존 작업하던 것을 통째로 메모리에 저장하고 다른 작업을 꺼내와서 수행하고, 또 저장하고 다른 작업을 번갈아가면서 이루어집니다.
  - 현재는 컴퓨터와 주변기기(I/O 장치, 저장장치 등) 모두 자체적으로 프로세서/컨트롤러 등이 다 내장되어 있으며, 운영체제는 드라이버를 통해 추상화된 함수 호출만으로 하드웨어 제어권을 장악하고 사용자를 대신하여 프로그램을 수행해주는 역할을 합니다.

* 운영체제의 역사
  - Multics는 MIT, 벨 연구소, 제너럴 일렉트릭이 공동 개발한 초기 시분할 운영체제이며 현대 운영체제의 개념(파일 시스템, 계층 구조, 멀티태스킹 등)에 아주 큰 영향을 미쳤습니다.
  - Unix는 최초로 고급 언어인 C 언어를 통해 만들어진 운영체제입니다. (벨 연구소에서 개발함)

* 이진 데이터로 어떻게 모든 데이터를 표현할 수 있나요?
  - 데이터에 대해 사전에 약속된 규약이 있기 때문에 숫자(정수, 실수) 외에도 문자, 이미지, 소리, 영상, 네트워크 통신 신호, 심지어 실행 파일 등 모든 것을 표현할 수 있습니다. (인코딩/디코딩)
  - 파일 시스템은 저장 장치 안에 파일이 어떤 위치에 저장되어 있는지에 대한 정보를 제공합니다.
  - 파일에는 각각 헤더가 존재하며 해당 파일이 어떤 용도인지, 어떤 방식으로 데이터가 저장되어 있는지 등의 정보가 기재되어 있습니다.
  - 네트워크 통신의 경우에도 헤더(경우에 따라서는 트레일러) 정보를 통해 뒤이어 들어오는 스트림의 길이와 내용에 대한 속성들을 파악할 수 있습니다.
  - 처음에는 항상 앞에서부터 접근하기 때문에 데이터에 대한 규약 정보(메타데이터라고도 함)는 헤더에 실려 있습니다.

### 컴퓨터 부팅 절차

1. 컴퓨터를 켜는 즉시 전원이 공급되기 시작함
2. 전원이 안정화되면 전원공급장치가 메인보드에게 'Power Good' 신호를 보냄
3. CPU는 미리 정해진 메모리 주소에 있는 명령어를 읽음
4. BIOS(Basic Input/Output System) 혹은 UEFI(Unified Extensible Firmware Interface) 실행
   - 여기서 POST (Power-On Self-Test) 실행: CPU, RAM, 그래픽 카드, 키보드, 마우스, 저장장치 점검하고 문제가 발생하면 Beep 발생 또는 오류 메시지 표시
5. 저장장치에 접근하여 MBR(Master Boot Record) 혹은 GPT(GUID Partition Table) 영역에 있는 부트 로더 실행
   - 부트 로더는 운영체제의 커널을 찾아 메모리에 적재하는 역할을 함
6. 커널이 로드되면 하드웨어 제어권을 장악한 후 시스템 구동에 필요한 자원을 할당하고 관리를 시작함 (드라이버 로드, 백그라운드 프로세스 및 서비스 구동)
7. 초기 프로세스(init)가 시작되고 Privilege 권한을 요구하지 않는 나머지 프로세스들을 구동함
   - CPU 자체적으로 보호 모드라는 것이 있어서 운영체제 커널의 Privilege 권한이 없는 경우 특정 명령어는 사용하지 못하도록 차단함
9. 부팅 완료 (로그인 대기)
